# 插件提交指南

## 📋 提交流程

### 1. 开发者注册
- GitHub 账号验证
- 开发者协议签署
- 身份信息验证

### 2. 插件提交
```bash
# 使用 CLI 工具提交
npx unihub-cli submit ./my-plugin.zip

# 或通过 Web 界面提交
https://plugins.unihub.dev/submit
```

### 3. 自动化检查
- [ ] 代码安全扫描
- [ ] 恶意软件检测
- [ ] 性能基准测试
- [ ] API 兼容性检查
- [ ] 文档完整性验证

### 4. 人工审核
- [ ] 功能完整性
- [ ] 用户体验
- [ ] 代码质量
- [ ] 安全性评估

### 5. 发布上架
- 自动部署到 CDN
- 插件商店更新
- 通知开发者

## 🏷️ 插件分类系统

### 官方认证 (Official)
- UniHub 团队开发
- 完全兼容保证
- 优先推荐

### 社区精选 (Featured)
- 高质量社区插件
- 经过严格审核
- 用户评分 4.5+ 

### 社区插件 (Community)
- 通过基础审核
- 社区贡献
- 用户自行评估

### 实验性 (Experimental)
- 新功能测试
- 可能不稳定
- 开发者预览

## 📊 质量评估标准

### 代码质量 (40%)
- 代码规范性
- 错误处理
- 性能优化
- 安全性

### 用户体验 (30%)
- 界面设计
- 交互流畅度
- 响应速度
- 易用性

### 文档完整性 (20%)
- README 文档
- API 文档
- 使用示例
- 更新日志

### 社区反馈 (10%)
- 用户评分
- 下载量
- 问题反馈
- 社区活跃度
